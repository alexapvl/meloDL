# meloDL

`meloDL` is a native macOS app for downloading and converting audio with a clean, settings-first workflow.

## Platform

- macOS (SwiftUI desktop app)

## What it does

- Accepts a media URL and downloads audio tracks.
- Supports configurable output format and quality.
- Supports playlist downloads with queue/status UI.
- Optionally embeds metadata and thumbnails (depending on audio format).
- Supports automatic update checks for bundled download binaries (`yt-dlp`, `ffmpeg`, `ffprobe`).

## Architecture overview

### Download pipeline

- `meloDL` uses bundled command-line tools:
  - [`yt-dlp`](https://github.com/yt-dlp/yt-dlp)
  - `ffmpeg` / `ffprobe`
- The app orchestrates these tools through `Process` execution in Swift.
- For playlist URLs, the app can prefetch entries and process multiple items.

### Binary updates (tool updates)

- `GitHubUpdateService` checks GitHub releases for:
  - `yt-dlp/yt-dlp`
  - `eugeneware/ffmpeg-static`
- Updated binaries are downloaded and replaced in Application Support.
- Version metadata is tracked in `Application Support/meloDL/versions.json`.

### App updates (Sparkle)

- The macOS app itself updates through [Sparkle](https://sparkle-project.org/).
- Sparkle reads an update feed (`appcast.xml`) from the URL configured in `Info.plist` (`SUFeedURL`).
- Each app release should publish:
  - A signed (and ideally notarized) release artifact (DMG/ZIP)
  - A corresponding `appcast.xml` entry generated by Sparkle tooling
